## 必须要做的功能

- 场景与事件模型：场景管理、事件 Schema、字段定义、样例报文、基础校验。
- 事件接入层：统一接入 HTTP / Beacon / SDK 三类上送方式，完成基础鉴权、来源识别、字段清洗、格式转换、字段标准化、公共字段补齐，并将标准事件写入 Kafka，异常事件进入 DLQ / error topic。
- 特征中心：Stream Feature、Lookup Feature、Derived Feature 三类都要有；其中流式特征先做 COUNT / SUM / MAX / LATEST / DISTINCT_COUNT(基础版)。
- 规则中心：表达式规则、优先级、动作类型、命中原因模板、启停状态。
- 策略中心：先把 FIRST_HIT 做稳；6 个月内最好补齐 SCORE_CARD，但优先级低于主链路打通。
- 发布中心：发布前校验、依赖分析、快照编译、版本号、发布记录、基础回滚。
- 运行时快照 + 热更新：Flink 通过配置流 + Broadcast State 接快照，做到 compile-before-activate，旧版本可保留/回滚。
- Flink 实时执行引擎：事件流消费、配置流消费、上下文构建、流式特征、Redis lookup、规则/策略执行、结果输出。
- 决策日志与追溯：可按 traceId / eventId 查结果，看到命中规则、特征快照、版本号、耗时。
- 仿真测试：输入模拟事件、指定场景/版本、展示特征值、命中链路、最终动作；这项其实是平台核心能力，不是可有可无的小工具。
- 最小可运行交付：Docker Compose、默认账号、默认场景、默认规则、默认仿真样例、演示链路。

## 推荐要做的功能

- 名单中心：黑白名单、导入导出、启停、同步 Redis；业务味道很强，做出来平台感会明显提升。
- 接入治理：来源系统管理、鉴权配置、来源启停、错误事件 / DLQ 查询、基础接入监控；做轻量版即可，但建议有。
- Dashboard / 基础监控：事件量、决策量、PASS/REVIEW/REJECT 占比、命中 Top 规则、P95 延迟、Kafka Lag、Checkpoint 成功率。
- 审计日志：谁改了规则、谁发布了版本、改了什么字段。
- 回放对比：用历史样例对比新旧版本差异，哪怕先做文件级 replay，也很值。
- 基础测试自动化：表达式执行器、规则执行器、策略执行器、快照编译器、仿真回归、1 条完整联调链路。
- 有限版 Groovy 扩展：可以做，但只能是少量高级规则/派生特征补位；必须沙箱化、预编译、缓存化，绝不能当主能力。
- 演示化包装：初始化数据、截图、README、快速启动文档、Demo 视频；这对 6 个月后的成果展示非常重要。

## 不推荐做的功能

- 不要做“任意脚本定义 keyBy/state/timer/window”。
- 不要先做 CEP/复杂序列规则。
- 不要先做拖拽式策略编排。
- 不要先做灰度发布、多环境推广、多租户 SaaS。
- 不要先做同步在线决策 API。
- 不要把机器学习平台一起卷进来。

## 总结

6 个月后至少应该能演示：配置场景/特征/规则/策略 -> 发布生成快照 -> 接入层接收业务事件并写入 Kafka -> Flink 热更新 -> Redis lookup -> 输出决策 -> 查询日志 -> 仿真验证 -> 回滚版本 -> 看监控大盘。

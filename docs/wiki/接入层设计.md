## 模块划分

```text
pulsix-access
  pulsix-ingest
  pulsix-sdk
```

- `pulsix-access`：数据接入层父模块
- `pulsix-ingest`：服务端统一接入器
- `pulsix-sdk`：业务后端高性能接入 SDK

## 模块功能

### `pulsix-ingest`
- 接收 `HTTP / Beacon / SDK` 上送事件
- 负责鉴权、基础校验、字段标准化、公共字段补齐
- 将标准事件写入 Kafka `standard topic`
- 将异常事件写入 `DLQ / error topic`

### `pulsix-sdk`
- 面向业务后端的高性能接入 SDK
- 基于 Netty 长连接向 `pulsix-ingest` 发送事件
- 提供批量发送、异步回调、重试与缓冲能力
- 只做轻量预处理，最终标准化仍由 `pulsix-ingest` 负责

## 数据链路

```text
1. 前端 / 埋点
Browser/App -> HTTP/Beacon -> Nginx -> pulsix-ingest -> standard topic -> Flink

2. 后端高性能接入
Business Service -> Pulsix SDK(Netty) -> pulsix-ingest -> standard topic -> Flink

3. 错误数据
pulsix-ingest -> DLQ / error topic
```

## 页面管理功能

**需要，但只做轻量治理能力。**

建议提供：
- 接入源管理
- 鉴权配置
- 错误事件 / DLQ
- 接入监控
- SDK 接入指引

不建议先做：
- 动态新增新协议
- 可视化接入编排
- 页面配置复杂标准化脚本

## 模块归属建议
- 不单独新建控制台系统
- 后端放在 `pulsix-module-risk` 的 `access/ingest` 子域
- 前端放在 `pulsix-ui` 的接入管理页面中

## 一句话总结
**接入层负责统一接入、鉴权、标准化和投递；控制台只做轻量治理，不做重型接入平台。**

# wiki instructions

When updating phase-1 engine/kernel docs, keep this split:

- `pulsix-engine-kernel-一期开发指南.md`: target scope and design boundary
- `pulsix-engine-kernel-一期进度.md`: actual current implementation status
- `pulsix-engine-kernel-一期-ai自动化开发方案.md`: AI execution protocol, task cards, milestone order

Rules:

- Keep docs concise and scannable.
- Do not duplicate large sections across the three files.
- If code reality changes, update the progress doc.
- If workflow changes, update the AI workflow doc.
- If phase-1 scope changes, update the development guide.

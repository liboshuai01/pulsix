# 项目一期定位

- 不做在线同步决策api, 仅提供异步风控链路, 风控动作仅支持 PASS, TAG_ONLY, REVIEW, REJECT 四种, 业务系统或下游系统需要具备异步处理风控结果的能力.
- redis只做名单,不做画像和热点特征. 页面提供名单管理, 支持手动同步数据到redis.
- 只做 FIRST_HIT, 忽略其他

## 页面开发

按“一个页面开发完成并验证后，再进入下一个页面”推进。统一使用 `TRADE_RISK` 样例做验收。

1. `risk` 菜单、路由、权限、页面骨架：菜单可见，页面可进入，刷新不丢路由。
2. 场景管理：完成 `scene_def` CRUD；可新增并查询 `TRADE_RISK`。
3. 事件模型：完成 `event_schema` / `event_field_def` CRUD；可维护 `trade` 事件并预览标准事件。
4. 实体类型：完成 `user/device/ip` 等实体配置；特征页可引用。
5. 名单中心：完成 `list_set` / `list_item` CRUD、导入、同步 Redis；可导入 `D_BLACK_001` 并同步成功。
6. 特征中心：完成 Stream / Lookup / Derived 三类特征；可建 `user_trade_cnt_5m`、`device_in_blacklist`、`high_amt_flag`。
7. 规则中心：完成 `rule_def` CRUD、表达式校验、动作配置；可录入 `TRADE_R001` ~ `TRADE_R005`。
8. 策略中心：完成 `policy_def` / `policy_rule_ref`；可配置 `TRADE_RISK_POLICY_FIRST_HIT` 的顺序和默认动作。
9. 发布管理：完成发布前校验、快照编译、发布；可生成 `scene_release` 的 `v1`。
10. 发布记录：可查看 `scene_release` 的版本、状态、发布人、发布时间。
11. 版本对比 / 回滚：可比较 `v1` / `v2` 差异，并回滚到旧版本。
12. 单条事件仿真：输入事件 JSON 可得到 `finalAction`、命中规则、特征快照。
13. 仿真用例：可保存 `E_TRADE_0002` ~ `E_TRADE_0005` 四条用例。
14. 仿真报告：执行用例后可生成报告，并区分通过 / 失败。
15. 决策日志：可按 `traceId / eventId / sceneCode` 查询决策结果。
16. 命中明细：可查看某次决策命中了哪些规则、命中值和命中原因。
17. 风险事件查询：可查询 `REVIEW / REJECT / TAG_ONLY` 结果。
18. 接入源管理：可维护 HTTP / SDK 来源和启停状态。
19. 鉴权配置：可为接入源配置鉴权参数，并验证合法 / 非法请求。
20. 错误治理：可查询解析失败、标准化失败、`DLQ` 事件。
21. 接入指引：可展示字段说明、样例报文、接入步骤。
22. 告警配置：可配置渠道、模板、静默、去重、限流。
23. 告警记录：可查询告警发送结果和静默 / 去重命中情况。
24. Dashboard：最后补首页指标；至少展示事件量、决策量、动作分布、命中 Top 规则、活跃版本、错误数。
